1. Use Redis for Real-Time Score Updates:
    When a score is updated, store it in Redis.
    Use Redis Pub/Sub to notify all connected WebSocket clients instantly.

2. Persist Data to MongoDB:
    After storing in Redis, update MongoDB for long-term persistence.
    You can use a delayed write approach where updates to MongoDB happen at intervals instead of every second.

3. Clients Fetch Data:
    Clients subscribe to Redis Pub/Sub for instant updates.
    If a client reconnects, fetch the last known score from Redis first (instead of MongoDB) for faster load times.
